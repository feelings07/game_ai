# Конвенция именования спрайтов

Все спрайты в проекте генерируются процедурно (Pillow/Python) и подключаются в Godot либо
статически (`.tscn`/`preload`), либо собираются в `SpriteFrames` на лету через
`scripts/anim_utils.gd`. Чтобы новые ассеты подхватывались существующим кодом без правок,
имена файлов должны следовать схеме ниже.

## 1. Персонажи (игрок, враги, босс) — покадровая анимация

Путь: `assets/sprites/<character_id>_<animation>_<N>.png`

- `character_id` — идентификатор персонажа: `player`, `enemy_goblin`, `enemy_skeleton`,
  `enemy_boss`, либо `id` нового врага из `data/enemies.json`.
- `animation` — название анимации:
  - `walk` — цикл ходьбы/блуждания, зацикленная. Сейчас 4 кадра (`0..3`), но код
    (`_frame_paths` в `castle_generator.gd`) принимает любое количество — если файлов больше
    или меньше 4, просто укажи весь список в `sprite_frames` (см. §3).
  - `attack` — анимация атаки, не зацикленная, проигрывается один раз и возвращает на `walk`.
    Может быть из 1 кадра (вспышка/удар) или нескольких.
- `N` — номер кадра, с нуля, без пропусков (`_0`, `_1`, `_2`, ...).

Примеры: `player_walk_0.png`, `enemy_goblin_attack_0.png`, `enemy_boss_walk_3.png`.

Размер канвы — квадратный, базовый размер персонажа (32 для игрока, 28 для обычных врагов,
48 для босса). Цикл ходьбы у существующих персонажей построен через squash/stretch одного
базового кадра (`make_walk_frames` в генераторных скриптах) — для нового персонажа можно либо
повторить этот приём, либо нарисовать кадры вручную с тем же базовым размером.

Игрок пока не имеет `attack`-анимации (атака мгновенная, без вспышки) — при необходимости
добавляется по той же схеме (`player_attack_0.png`, ...).

## 2. Предметы — два спрайта на каждый id

У каждого предмета из `data/items.json` должно быть **два** файла:

| Назначение | Путь | Поле в JSON |
|---|---|---|
| Иконка в инвентаре (плоская пиктограмма, с тёмной подложкой-панелью) | `assets/sprites/items/<item_id>.png` | `"icon"` |
| Спрайт на полу (без подложки, прозрачный фон — крутится при показе на земле) | `assets/sprites/items/<item_id>_world.png` | `"world_icon"` |

`item_id` — это `id` предмета из `data/items.json` (например `sword_iron`, `wand_fire`).
Канва — 32×32. Иконка инвентаря рисуется с фоном-панелью (см. `panel()` в генераторных
скриптах) и сравнительной подсветкой статов в тултипе. World-спрайт — то же изображение
предмета, но без панели: его крутит `scripts/world_item.gd` вокруг своей оси (`rotation +=
delta`), подложка тут не нужна и будет мешать.

Если у предмета нет своего `world_icon` в JSON, код подставит `icon` как запасной вариант —
но для нового предмета лучше явно нарисовать оба.

## 3. Как это подключается в код (для справки, не для ручного редактирования)

- `data/enemies.json` → поля `"sprite_frames"` (кадры `walk`) и `"attack_frames"` (кадры
  `attack`) — массивы путей. Если поле не указано, код подставит дефолтный список по
  `<character_id>_walk_N.png` / `<character_id>_attack_N.png` (см. `_frame_paths` в
  `castle_generator.gd`).
- `data/items.json` → поля `"icon"` и `"world_icon"`.
- `scripts/anim_utils.gd::build_sprite_frames(animations: Dictionary)` собирает `SpriteFrames`
  из словаря `{"walk": [...], "attack": [...]}` — добавление новой анимации (например
  `"hurt"`) не требует правок этого файла, только передачи нового ключа при вызове.

## 4. Прочие спрайты (не по этой схеме, фиксированные пути)

- Снаряды: `assets/sprites/proj_<имя>.png` (`proj_fire`, `proj_ice`, `proj_lightning`),
  `assets/sprites/arrow.png`.
- Тайлы: `assets/tilesets/tile_<имя>.png`.
- Прочие мировые объекты: `assets/sprites/<имя>.png` (`chest_closed`, `chest_open`,
  `trap_spike`, `shadow_small`, `shadow_large`).

## 5. Стиль отрисовки (чтобы новые ассеты не выбивались)

- Вид строго сверху (top-down), персонажи — силуэт-капля с "носом" в сторону взгляда.
- Лёгкий объём: на каждой крупной форме — полупрозрачный светлый эллипс сверху-слева
  (хайлайт) и тёмный эллипс снизу-справа (тень), см. `shade()` в генераторных скриптах
  персонажей и `add_bevel()` для иконок предметов.
- Тень под ногами/объектом — отдельный некрутящийся/неанимированный спрайт
  (`shadow_small.png` / `shadow_large.png`), а не часть кадра анимации.
